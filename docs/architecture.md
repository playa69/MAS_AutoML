# Архитектура МАС для AutoML

## Высокоуровневый обзор

Система состоит из трёх основных агентов и набора сервисов, обеспечивающих выполнение гибридных AutoML-пайплайнов:

- **PlannerAgent** — анализирует задачу, выбирает подходящие AutoML-фреймворки и формирует последовательность шагов.
- **ResearchAgent** — выполняет поиск в научных источниках и пополняет базу знаний новыми моделями и техникой.
- **ExecutorAgent** — оркестрирует запуск шагов пайплайна, агрегирует результаты и формирует финальные метрики.

Оркестратор `PipelineOrchestrator` отвечает за вызов адаптеров конкретных AutoML-фреймворков. Каждый адаптер реализует асинхронный метод `execute`, который запускает фреймворк с заданными параметрами и возвращает `FrameworkResult`.

## Поток данных

1. Пользователь или внешний сервис формирует запрос (описание задачи, ограничения, данные).
2. `PlannerAgent` строит план пайплайна (например, предобработка в AutoGluon → обучение в Auto-sklearn → ансамбль в FEDOT).
3. При необходимости `ResearchAgent` дополняет план новыми шагами, найденными в актуальных публикациях.
4. `ExecutorAgent` преобразует план в `HybridPipeline` и передаёт его оркестратору.
5. Оркестратор последовательно (или параллельно) вызывает адаптеры фреймворков и агрегирует результаты.
6. Сводный отчёт сравнивается с базовыми результатами каждого фреймворка и возвращается пользователю.

## Источники данных и знания

- Датасеты: AMLB benchmark (классификация и регрессия).
- Публикации: arXiv, OpenReview, корпоративные знания.
- API: LLM-провайдеры, поисковые движки, репозитории кода.

## Следующие шаги реализации

- Реализовать адаптеры для AutoGluon, Auto-sklearn 2.0 и FEDOT с реальными вызовами.
- Добавить управление ресурсами (GPU/CPU), очередь задач и мониторинг.
- Интегрировать MLflow или аналогичный инструмент для отслеживания экспериментов.
- Подготовить конфигурацию для запуска пайплайнов в Docker/Kubernetes.
